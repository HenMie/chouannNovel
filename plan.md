# 前端页面优化计划 (Optimization Plan)

基于对当前代码库的分析，以下是对各个页面及组件的优化建议和计划。目标是提升用户体验 (UX)、统一视觉风格 (UI)，并增强功能的易用性。

## 1. 已完成优化 (Completed)

### Sidebar & Header
- [x] **Sidebar**: 添加搜索功能，集成 `AlertDialog`，优化折叠/展开交互。
- [x] **Header**: 集成面包屑导航 (Breadcrumb) 和主题切换。

### 核心页面 (Core Pages)
- [x] **首页**: 升级 Hero 区域，优化项目卡片（封面/头像、元数据、菜单），添加排序和导入占位。
- [x] **项目详情页**: 添加概览仪表盘，优化工作流列表展示，增强设定库入口。
- [x] **设置页面**: 重构为垂直导航布局，改进 AI 提供商卡片（连接测试、表单体验）。

### 功能页面 (Feature Pages)
- [x] **设定库**: 
    - 编辑改为侧边抽屉 (`Sheet`)。
    - 文本编辑器增加 Markdown 工具栏。
    - 提示词模板编辑器增加变量插入辅助。
    - 添加搜索功能。
- [x] **执行历史**: 
    - 优化列表密度和状态展示。
    - 增加全屏详情查看、复制和导出 (TXT/MD) 功能。

---

## 2. 待办开发计划 (Upcoming)

### 后续功能增强
- [x] **Markdown 渲染**: 引入 `react-markdown` 库，在执行历史和流式输出中提供更好的 Markdown 渲染支持（代码高亮、表格等）。
- [x] **后端数据支持**: 
    - 实现真实的统计接口（字数统计、运行次数等），替换前端 Mock 数据。
    - 优化搜索接口，支持服务端过滤。

### 组件库与样式统一
- [x] **Loading 状态**: 定义统一的页面级和局部 Loading 组件（Skeleton 骨架屏），替换现有的简单 Spinner。
- [x] **空状态 (Empty State)**: 继续完善所有列表的空状态插图和引导文案。
- [x] **Toast 反馈**: 规范化 Toast 通知的样式和持续时间，确保成功/失败/警告的语义明确。
- [x] **字体与排版**: 统一全站的字体层级 (Typography)，特别是标题、正文和代码块的样式。

### 性能优化
- [ ] **列表虚拟化**: 对于可能包含大量项目或日志的列表，引入虚拟滚动 (Virtual Scroll) 以提升性能。
- [x] **按需加载**: 确保图表库或大型组件（如 Markdown 渲染器）是按需加载的。
